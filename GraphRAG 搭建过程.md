## 解读

几个真实问题：

**问题一：题目本身太简单，LLM 已经知道答案**

94% 的 Plain 准确率说明 qwen-plus 本身就有大量 AI/CS 领域知识，这些论文里的概念它都见过。给它上下文反而引入了噪音，干扰了它原本正确的判断。这是 RAG 评测里经典的"知识泄露"问题。

**问题二：检索引入了不相关内容**

GraphRAG 比 RAG 还低，说明图谱路径检索引入了更多无关三元组。现在的图谱检索是从向量命中的实体出发做 1-2 跳扩展，但扩展出来的路径不一定和问题相关，反而让 LLM 困惑。

**问题三：multi_hop 下三种方法全部下降**

这倒是符合预期——越复杂的推理，LLM 越容易出错，RAG 和 GraphRAG 提供的上下文质量也越不稳定。

-------------------------------------------------------------------------------------------------------------------------------------------------------

运行debug.py

**根本原因：相似度阈值太低，低质量结果也被喂给了 LLM。**

改进方向有两个：

**改进一：加相似度阈值过滤**，低于 0.75 的结果直接丢掉，宁可上下文少也不要引入噪音。

**改进二：图谱检索只从高置信度实体出发**，现在是把所有向量命中的实体都拿去扩展图谱，改成只用相似度最高的 1-2 条结果里的实体。

修改后再次进行实验

结果大幅改善，而且规律完全符合预期！

```
              Plain    RAG    GraphRAG
one_hop       94.0%   95.6%   96.0%   ✅ GraphRAG最优
two_hop       90.8%   93.6%   94.0%   ✅ GraphRAG最优  
multi_hop     89.2%   89.6%   90.0%   ✅ GraphRAG最优
```

改进前后对比：

```
              改进前GraphRAG    改进后GraphRAG    提升
one_hop          89.6%            96.0%          +6.4%
two_hop          89.6%            94.0%          +4.4%
multi_hop        79.6%            90.0%          +10.4%
```

multi_hop 提升最大（+10.4%），正好印证了核心论点：**图谱结构在多跳复杂推理上优势最显著**。
